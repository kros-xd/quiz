  1. Name a real-life situation where you would need to use an `if` statement.
  2. A real life situation would be if I'm hungry, get food. Else, don't get food.
  3. Explain the difference between `==` and `===`. Why is strict equality usually better?
  4. The loose equality '==' in JavaScript implicitly converts data types to match, so that the values can be evaluated. However, strict equality '===' evaluates if the data types are the same AND if the values are the same.
  5. Give an example (in your own words) of when you would use `&&` (AND) vs `||` (OR).
  6. I like to use logical operators when I need all conditions to be true or at least one has to be true. For example if we had a for loop that counts cards and its points, but we don't want it to go over a certain score, we can do this: for (let i = 0; i < cardSize && !(currentScore > 21); i++) {...} This will iterate over our hypothetical cards so long as our currentScore isn't greater than 21. If it goes over it will terminate the loop, or we reach the end of available cards. For the logical operator or ||, only one expression needs to be true. Let's say we want to keep playing a game as long as the user doesn't quit or if we dont meet the score requirement. We can do this: while (continue != 'q' || score >= 21) {...} This means that the game will continue while the user hasn't pressed q or if the score isn't greater than or equal to 21.
  7. What does the `!` (NOT) operator do? Give an example of a situation you would use it.
  8. The not operator (!), flips the result of a condition. For example if we want to have a game run while no one has one yet: let won = false; while (!won) { ... } The not operator does the opposite condition of the boolean. since we have 'won' set to false, using the not operator turns it into a truthy value.
